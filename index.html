git是什么？

假如多个人共同开发一个项目，怎么确保代码的同步？假如你修改了别人的代码或者别人修改了你的代码，很容易出现冲突的情况。
于是git诞生了，如果你找工作不会用git，这是致命的不足。因为没有那个公司不用它！

github仓库

github=git+hub，即git仓库。git仓库有很多，github或者国内的码云，它是一个云仓库，相当于一个中介，我把代码上传上去，你把代码拉下来，这样我们就能共享代码。
而且仓库自己带有解决冲突的机制，帮助我们解决代码冲突！

初识maven

发表于 2017-08-20 | 分类于 maven |

前言

很久之前用过maven开发项目，但这几个月一直在公司写node.js，很少碰Java，为了不生疏，就写一系列的博客加强记忆，顺便教下小白（PS：我自己也是小白）。

为什么用maven？

用Java框架（SSH、SSM）开发过Web项目的都知道，随着项目越做越大，在WEB-INF/lib目录下放的Jar包也越来越多，当你需要增加或删除一个Jar包，全靠人工记忆，非常不便于管理项目。
而且各个Jar包是否有版本冲突情况，让人头疼。

maven改进

当你需要一个Jar包时，你只需要在pom.xml中申明它，然后更新工程，Jar包会自动下载下来，一般在C:/Users/xx/.m2/repository文件夹中（看你具体怎么配置的）。
当项目编译时，Jar包才会被导入到编译后的项目目录中，随着项目发布。

前言

要理解RTTI（运行时类型信息）在Java中的工作原理，必须知道类型信息是如何表示的。

类加载机制

每个类都有一个Class对象，每当编译了一个类，就会产生一个Class对象，保存在同名的.class文件中。
当程序运行时，虚拟机把描述类的数据从class文件中加载到内存，并对数据进行校验，转换解析和初始化，最终形成可以被虚拟机直接使用的Java类型，这就是虚拟机的类加载机制。

垃圾内存回收

发表于 2017-07-30 | 分类于 Java |

前言

Java语言是支持内存自动回收的，不像C++中new和delete要成对出现以避免内存泄露。

找出回收的内存

  1. 标记计数法：
    给内存中的对象给做上标记，对象被引用一次，计数就加1，引用被释放了，计数就减1，当这个计数为0的时候，这个对象就可以被回收了。
    缺点：无法解决循环引用的问题。
  2. 根搜索算法：
    从内存中有效的对象出发，搜索所有的可达对象，这样剩下的那些对象就是需要被回收的。

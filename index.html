网页显示

发表于 2017-07-24 | 分类于 HTTP

前言

打开一个浏览器，输入www.baidu.com， 回车，就会显示出网页，那么里面到底发生了什么？

  1. 结果可能会重定向，例如你输入baidu.com，浏览器会自动转化为https://www.baidu.com/。
  2. 浏览器自身DNS缓存（要拿到网址对应的IP地址嘛）。
  3. 操作系统自身的DNS缓存，前提是浏览器缓存已经失效或者过期。
  4. 本地host文件（localhost对应的IP 127.0.0.1就存在这里）。
  5. 还没有，那要开始对DNS服务器发起一个请求了，先到达路由器，如果路由器有，就不用找了。没有，就对ISP（互联网服务提供商，中国移动等xx）发起请求。
  6. 像树一样，先找com域名服务器，再找baidu，找到了就OK，否则提示用户网址不存在。
  7. 终于找到服务器了，开始发起一个HTTP请求，三次握手后，开始通信，服务器把网页代码都发送给浏览器。
  8. 浏览器拿到代码后，开始渲染，解析，最终把网页呈现出来。对于网页中用到的css、图片等资源，还要通过以上的步骤才能拿到。

String,StringBuilder,StringBuffer

发表于 2017-07-24 | 分类于 Java

String

String是放在常量池中的常量，所谓常量，就是这个对象一旦产生，它的值就不会变。有人疑问：不是有String a=a+”xx”这样的写法吗，这是因为a在加上”xx”后，已经不是原来的a了，a在加之后又变成了一个新的对象，只不过名字也叫a。

可以看出String的的存储实际上就是一个final的数组（该数组不允许再指向别的地址，但数组的值可以改变），所以String的地址是无法改变的，每次更新后都是一个新的String对象。

String类

发表于 2017-07-24 | 分类于 Java

前言

String类是Java中的常用类，面试也会经常被问到。下面我就一个问题进行切入：
问题：String a=new String(“a”) ，这条语句执行后创建了几个对象？

解答

  1. String的存储：String是保存在常量池中的，何为常量池，这里我又要拉扯到享元模式，多人共享一个东西有什么好处？资源不会被浪费啊，看看共享单车就是这个道理。字符串常量池一句话：如果有，我就从池中拿，如果没有，我就新建，然后放入池中。所以当程序看到”a”时，常量池以前是没有”a”的，那么常量池中就会多一个”a”，这是1个对象。
  2. 程序看到new关键字，这个时候在堆内存中新建了一个对象a，这是第2个对象。

finalize与finally

发表于 2017-07-24 | 分类于 Java

前言

众所周知，Java语言除了8种基本的数据类型以外，处处皆对象，Java中所有的类都是继承于Object类（包括你自己写的），而finalize()是Object类中的一个方法，这个方法在垃圾内存回收中被用到。

finalize

当一个对象A的内存要被回收时，会调用其finalize()方法，如果A对象没有该方法，对不起，这块内存会直接被释放掉，如果在A这个方法内，A对象又被新的对象引用，那么A对象将重生了！即A对象的内存不会被释放。
注意：不要把finalize()与C++中的析构函数相提并论，因为它们不是同一个玩意，一个对象被析构了，它不一定会被内存回收。

代理模式

发表于 2017-07-24 | 分类于 设计模式

个人理解

老板要做一件事情，但是他不用亲自去做，而是交给秘书代理完成。

好处

秘书代理完成的过程中可以添加自己的行为，例如完成前，他可以做xx事情，完成后，他可以再做xx事情。代理模式常用于给对象添加新的功能（而不改变原有的对象）。

AOP面向切面编程

Spring中的注解@Transaction也是用到了代理模式，把你的函数上加上一个注解@Transaction，你的函数就变成了事务性的，原子性的。为什么，因为Spring帮你做了，在调用函数前，打开一个事务，结束函数后，提交一个事务。总之，这个设计模式比较重要，以后我还会写一下JDK的动态代理，也是用到了代理模式。

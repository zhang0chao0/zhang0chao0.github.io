事务隔离级别

发表于 2017-08-16 | 分类于 数据库 |

前言

数据库事务的隔离级别有4个，由低到高依次为Read uncommitted（读未提交）、Read committed（读已提交）、Repeatable read（重复读）、Serializable（序列化）。

读未提交

A，B共有一张银行卡，A向卡中打入1000元，但事务未提交，B查看卡发现多了1000元；
但A发现打错了，回滚了事务，B再查看就发现钱没有了。
B之前读了未提交的事务，出现了脏读。

读已提交

A发现卡中有1000元，准备消费，此时B转走了1000块并提交了事务；
A结账时发现钱已经没有了。
A读已提交的事务，说明读已提交的事务不能重复读，否则会出错。

重复读

MySQL的默认级别是重复读，重复读可能出现幻读。
A读卡中有1000元，当A再次读时，钱已经被B转走了并提交了事务，此时A会产生一种读错数据的幻觉。

序列化

Serializable是最高的事务隔离级别，同时代价也花费最高，性能很低，一般很少使用，在该级别下，事务顺序执行，不仅可以避免脏读、不可重复读，还避免了幻读。

总结

脏读 不可重复读 幻读
Read uncommitted √ √ √
Read committed x √ √
Repeatable read x x √
Serializable x x x

回顾

数据库的四大特性：

  1. 原子性：
    指事务包含的所有操作要么全部成功，要么全部失败回滚，不可分割。
  2. 一致性：
    是指事务必须使数据库从一个一致性状态变换到另一个一致性状态，也就是说一个事务执行之前和执行之后都必须处于一致性状态。例如A->B转账100，A账户少了100，B账户一定要多100。
  3. 隔离性：
    隔离性是当多个用户并发访问数据库时，比如操作同一张表时，数据库为每一个用户开启的事务，不能被其他事务的操作所干扰，多个并发事务之间要相互隔离。
  4. 持久性：
    持久性是指一个事务一旦被提交了，那么对数据库中的数据的改变就是永久性的，即便是在数据库系统遇到故障的情况下也不会丢失提交事务的操作。

String类

前言

String类是Java中的常用类，面试也会经常被问到。下面我就一个问题进行切入：
问题：String a=new String(“a”) ，这条语句执行后创建了几个对象？

解答

  1. String的存储：String是保存在常量池中的，何为常量池，这里我又要拉扯到享元模式，多人共享一个东西有什么好处？资源不会被浪费啊，看看共享单车就是这个道理。字符串常量池一句话：如果有，我就从池中拿，如果没有，我就新建，然后放入池中。所以当程序看到”a”时，常量池以前是没有”a”的，那么常量池中就会多一个”a”，这是1个对象。
  2. 程序看到new关键字，这个时候在堆内存中新建了一个对象a，这是第2个对象。

String的比较

经常会有这种比较，看得脑阔疼。但是只要明白了==是比较啥的，就迎刃而解了。==用于比较：

  1. 8种基本数据类型的大小。
  2. 两个变量是否同一块内存。很明显，两次new出来的内存能一样吗？一样的概率就是你中彩票的概率了，还有就是堆内存和常量池中的内存，肯定是不一样的！结果的话大家可以自己在本地下跑一下，加深印象。

引申问题

问题：String a=”a”+”b”+”c”; 内存中会出现几个对象？
解答：第一眼会以为是3个，其实就错了，其实就1个对象”abc”在常量池中。为毛？因为编译器很聪明，它在编译的时候就算出”a”+”b”+”c”的值为”abc”了，所以编译器看到的就是”abc”，那么常量池中就1个对象了。

String,StringBuilder,StringBuffer

发表于 2017-07-24 | 分类于 Java

String

String是放在常量池中的常量，所谓常量，就是这个对象一旦产生，它的值就不会变。有人疑问：不是有String a=a+”xx”这样的写法吗，这是因为a在加上”xx”后，已经不是原来的a了，a在加之后又变成了一个新的对象，只不过名字也叫a。

可以看出String的的存储实际上就是一个final的数组（该数组不允许再指向别的地址，但数组的值可以改变），所以String的地址是无法改变的，每次更新后都是一个新的String对象。

StringBuilder

为了解决String重复更新而消耗内存的情况，StringBuilder产生了。StringBuilder对字符串的操作都是基于原内存的，原内存不会改变，例如StringBuilder的append()方法，可以原有字符串的后面添加一段字符串，而不用像String一样重复赋值了。StringBuilder继承于AbstractStringBuilder：

可以看出并没有final关键字修饰。

StringBuffer

这个跟StringBuilder一样继承于AbstractStringBuilder，唯一的不同之处，就是加了synchronized。

可以看到，每个关键的操作函数都加了synchronized关键字修饰，可以看出这是一个线程安全的操作，非静态方法被synchronized修饰后，当有多个线程操作同一个StringBuffer对象时，它们只能有一个操作append方法。所以说StringBuffer是线程安全的StringBuilder，那么在效率上肯定不如StringBuilder了。

synchronized

关于synchronized的加锁，在静态方法和非静态方法上加是有区别的。静态方法上加会影响这个类的所有对象，而非静态上加只是影响其中某一个对象。具体看看static关键字的用法，不多解释了。

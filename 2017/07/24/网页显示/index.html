网页显示

发表于 2017-07-24 | 分类于 HTTP

前言

打开一个浏览器，输入www.baidu.com， 回车，就会显示出网页，那么里面到底发生了什么？

  1. 结果可能会重定向，例如你输入baidu.com，浏览器会自动转化为https://www.baidu.com/。
  2. 浏览器自身DNS缓存（要拿到网址对应的IP地址嘛）。
  3. 操作系统自身的DNS缓存，前提是浏览器缓存已经失效或者过期。
  4. 本地host文件（localhost对应的IP 127.0.0.1就存在这里）。
  5. 还没有，那要开始对DNS服务器发起一个请求了，先到达路由器，如果路由器有，就不用找了。没有，就对ISP（互联网服务提供商，中国移动等xx）发起请求。
  6. 像树一样，先找com域名服务器，再找baidu，找到了就OK，否则提示用户网址不存在。
  7. 终于找到服务器了，开始发起一个HTTP请求，三次握手后，开始通信，服务器把网页代码都发送给浏览器。
  8. 浏览器拿到代码后，开始渲染，解析，最终把网页呈现出来。对于网页中用到的css、图片等资源，还要通过以上的步骤才能拿到。

CDN服务器

从步骤5后，请求到达DNS负载均衡系统：

  1. 智能DNS负载均衡系统解析域名，把对用户响应速度最快的IP节点返回给用户。
  2. 用户向该IP节点（CDN服务器）发出请求。
  3. 由于是第一次访问，CDN服务器会向原web站点请求，并缓存内容。
  4. 然后进入步骤7。
    没错，其实干了这么多事，当然我还有总结的不全的地方，欢迎指出。

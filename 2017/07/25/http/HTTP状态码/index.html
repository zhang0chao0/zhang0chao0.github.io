HTTP状态码

发表于 2017-07-25 | 分类于 HTTP |

前言

本文总结于网络，总结一下HTTP返回的状态码。

基本

  • 100-199 用于指定客户端应相应的某些动作。
  • 200-299 用于表示请求成功。
  • 300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息。
  • 400-499 用于指出客户端的错误。
  • 500-599 用于支持服务器错误。

细节

100

客户端询问服务器是否在后续的请求中可以发送附件，会在请求头中加上100-continue，如果服务器同意，返回status 100，如果不同意，则返回417 (Expectation Failed)。

101

(Switching Protocols/转换协议)不太明白。

200

最熟悉了，代表OK、正确。

201

(SC_CREATED)表示服务器在请求的响应中建立了新文档。

202

(SC_ACCEPTED)告诉客户端请求正在被执行，但还没有处理完。

203

(SC_NON_AUTHORITATIVE_INFORMATION)是表示文档被正常的返回，但是由于正在使用的是文档副本所以某些响应头信息可能不正确。

204

无文档，(SC_NO_CONTENT)确保浏览器继续显示先前的文档。

205

(SC_RESET_CONTENT)的意思是虽然没有新文档但浏览器要重置文档显示。这个状态码用于强迫浏览器清除表单域。

300

(SC_MULTIPLE_CHOICES)表示被请求的文档可以在多个地方找到，并将在返回的文档中列出来。如果服务器有首选设置，首选项将会被列于定位响应头信息中。

301

(SC_MOVED_PERMANENTLY)状态是指所请求的文档在别的地方。文档新的URL会在定位响应头信息中给出，浏览器会自动连接到新的URL。

302

与301有些类似，只是定位头信息中所给的URL应被理解为临时交换地址而不是永久的。

303

如果之前用post方式请求文档，那么之后要用get方式请求。（不知道理解的对不对，可能有错误）。

304

熟悉，表示资源没有被修改，如果你请求一个接口两次就会返回这个。

305

(SC_USE_PROXY)表示所请求的文档要通过定位头信息中的代理服务器获得。

307

(Temporary Redirect/临时重定向)在响应为303时按照GET和POST请求转向，在307响应时则按照GET请求转向而不是POST请求。

400

错误请求（语法错误）。

401

未授权请求，请求时要在头部加上授权信息。

403

禁止访问，(SC_FORBIDDEN)的意思是除非拥有授权否则服务器拒绝提供所请求的资源。这个状态经常会由于服务器上的损坏文件或目录许可而引起。

404

很熟悉，未找到页面。一般是请求的url错误。

405

(SC_METHOD_NOT_ALLOWED)指出请求方法(GET, POST, HEAD, PUT, DELETE, 等)对某些特定的资源不允许使用。

406

无法访问，(SC_NOT_ACCEPTABLE)表示请求资源的MIME类型与客户端中Accept头信息中指定的类型不一致。

407

这个状态用于代理服务器，该状态指出客户端必须通过代理服务器的认证。

408

请求超时，time out。

409

请求冲突，(SC_CONFLICT)状态常被用于试图上传版本不正确的文件时。

410

请求文件已经被移走了，不存在了。

411

(SC_LENGTH_REQUIRED)表示服务器不能处理请求（假设为带有附件的POST请求），除非客户端发送Content-Length头信息指出发送给服务器的数据的大小。

412

(SC_PRECONDITION_FAILED)状态指出请求头信息中的某些先决条件是错误的。

413

请求实体过大，(SC_REQUEST_ENTITY_TOO_LARGE)告诉客户端现在所请求的文档比服务器现在想要处理的要大。如果服务器认为能够过一段时间处理，则会包含一个Retry-After的响应头信息。

414

请求url过长。

415

不支持的媒体格式。

416

(Requested Range Not Satisfiable/请求范围无法满足)

417

(Expectation Failed/期望失败)，参考status 100。

500

服务器内部错误，服务器端代码写错了。

501

(SC_NOT_IMPLEMENTED)状态告诉客户端服务器不支持请求中要求的功能。例如，客户端执行了如PUT这样的服务器并不支持的命令。

502

错误的网关。

503

(SC_SERVICE_UNAVAILABLE)表示服务器由于在维护或已经超载而无法响应。例如，如果某些线程或数据库连接池已经没有空闲则servlet会返回这个头信息。服务器可提供一个Retry-After头信息告诉客户端什么时候可以在试一次。

504

(Gateway Timeout/网关超时)

505

不支持的HTTP协议版本。

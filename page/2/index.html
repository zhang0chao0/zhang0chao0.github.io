builder模式

发表于 2017-07-24 | 分类于 设计模式

个人理解

我要写一个文件，文件分为xml文件和txt文件，文件都有的特性：文件头header，文件内容body，文件为footer。我肯定要用一个抽象的builder接口（有builder文件的方法），然后用具体的实现类去实现builder。还要有一个director类来控制整个builde流程。

好处

使用builder模式，所有的控制权都在director上，一个大系统有很多零件，director可以决定哪个零件要，哪个零件不要，以及零件的组装顺序。万一有一天系统中的某个零件不要了或者零件的安装顺序变了，只用修改director类，其它都不用变。

设计模式概述

发表于 2017-07-24 | 分类于 设计模式

前言

本文总结于网络。软件设计的核心在于提高软件的可复用性和可维护性。设计模式就用于解决这一问题。

设计原则

  • 开闭原则，简称OCP。即软件实体对扩展是开放的，对修改是闭合的。
  • 里氏替换原则，简称LSP。即任何父类出现的地方都可以被子类代替。
  • 依赖倒转原则，简称DIP。即要依赖于抽象而不要依赖于具体实现。
  • 接口隔离原则，简称ISP。即为客户端尽可能提供小的、单独的接口，而不是提供更大的接口。
  • 组合/聚合复用原则，简称CARP。即尽量使用组合、聚合来达到复用的目的，而不是利用继承。
  • 迪米特法则，简称LOD，讲的是一个软件实体应当与尽可能少的其他软件实体发生相互作用。

分类

  • 创建模式
  • 结构模式
  • 行为模式

快速排序

发表于 2017-07-24 | 分类于 排序算法

前言

好几个同学面试都问到了快速排序，个人觉得这也是面试中的常考点，所以我也总结一下，快速排序是基于分治算法的。

基本步骤如下：

  1. 选择一个基准元素，通常选择第一个元素或者最后一个元素；
  2. 通过一趟排序讲待排序的记录分割成独立的两部分，其中一部分记录的元素值均比基准元素值小。另一部分记录的元素值比基准值大；
  3. 此时基准元素在其排好序后的正确位置；
  4. 然后分别对这两部分记录用同样的方法继续进行排序，直到整个序列有序为止。

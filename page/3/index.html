前言

类加载机制

垃圾内存回收

发表于 2017-07-30 | 分类于 Java |

前言

Java语言是支持内存自动回收的，不像C++中new和delete要成对出现以避免内存泄露。

找出回收的内存

  1. 标记计数法：
    给内存中的对象给做上标记，对象被引用一次，计数就加1，引用被释放了，计数就减1，当这个计数为0的时候，这个对象就可以被回收了。
    缺点：无法解决循环引用的问题。
  2. 根搜索算法：
    从内存中有效的对象出发，搜索所有的可达对象，这样剩下的那些对象就是需要被回收的。

JVM

发表于 2017-07-26 | 分类于 Java |

前言

JVM模型定义了一些内存区域，有的在程序运行时产生，在程序结束时销毁，称之为全局区域，有的在线程运行时创建，线程结束时销毁，称之为 线程私有的内存区域。

内存结构分区图：



程序计数器

程序计数器是一块比较小的内存区域，用来存放当前线程执行字节码的行号，许多基础的功能例如分支、循环都要依赖它。
由于Java多线程是轮流切换并分配处理器执行时间的方式来实现的，所以在一个确定的时刻，CPU只能处理一个线程。为了每个线程运行互不干扰，所以程序计数器是线程私有的。

PS：

  • 如果线程正在执行的是一个Java方法，这个计数器记录的是正在执行的虚拟机字节码指令的地址；
  • 如果正在执行的是Natvie方法，这个计数器值则为空。

HTTP状态码

发表于 2017-07-25 | 分类于 HTTP |

前言

本文总结于网络，总结一下HTTP返回的状态码。

基本

  • 100-199 用于指定客户端应相应的某些动作。
  • 200-299 用于表示请求成功。
  • 300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息。
  • 400-499 用于指出客户端的错误。
  • 500-599 用于支持服务器错误。

细节

100

客户端询问服务器是否在后续的请求中可以发送附件，会在请求头中加上100-continue，如果服务器同意，返回status 100，如果不同意，则返回417 (Expectation Failed)。

101

(Switching Protocols/转换协议)不太明白。

200

最熟悉了，代表OK、正确。

dynamodb中scan数据不全

发表于 2017-08-26 | 分类于 node |

前言

来自一个真实测试的bug，在公司实习主要用node开发，数据库是用的dynamodb。这个数据库跟mongodb一样，也属于no sql数据库。但是用的人相对较少，网上的博客也不多，学习只能看官方文档。

bug

起源于一条记录，在log（文件）中可以看到它，但是查询日志（从数据库中查）时却找不到它。经过排查发现，数据库中有这个数据，但是没查到！

过程

  • 是不是查询语句的问题？换了几种查询方式，没用，还是一样。
  • 是不是数据本身问题？把若干数据人工输入到本地数据库，本地测试，呵呵，本地没问题。

解决

最后没办法了，问了公司一位老司机（技术某某大牛吧）。指出了两点待改进的错误：

  1. 数据库查询scan的效率极低，每次都会扫描全表，如果一个增长数据很快的数据库，经常用scan的查询效率是极低的，如果你用了scan，只能说明你的表结构设置的不对。
  2. 当scan的结果中有LastEvaluatedKey字段时，说明数据查询并没有结束，还要利用这个字段递归查询。

git入门

发表于 2017-08-21 | 分类于 git |

前言

为了照顾刚入门的小白，搭建博客起来还是有点困难，特写两篇git入门和node入门来帮助他们。

git是什么？

假如多个人共同开发一个项目，怎么确保代码的同步？假如你修改了别人的代码或者别人修改了你的代码，很容易出现冲突的情况。
于是git诞生了，如果你找工作不会用git，这是致命的不足。因为没有那个公司不用它！

github仓库

github=git+hub，即git仓库。git仓库有很多，github或者国内的码市，它是一个云仓库，相当于一个中介，我把代码上传上去，你把代码拉下来，这样我们就能共享代码。
而且仓库自己带有解决冲突的机制，帮助我们解决代码冲突！

前言

周末一个线上测试的脚本跑了两天，一直报错：

很明显，这是一个http的错误。

联想

之前也遇到过这个问题，原因是http请求端口号写错了，导致不能发起连接。

调试

在本地测试后（主要是打log），发现获取数据正在进行时，脚本就返回了这个错误。因为获取数据的时间太长了，http请求的默认超时时间是2分钟，所以导致http请求超时了。
解决思路：自定义http请求的超时时间。

初识maven

发表于 2017-08-20 | 分类于 maven |

前言

很久之前用过maven开发项目，但这几个月一直在公司写node.js，很少碰Java，为了不生疏，就写一系列的博客加强记忆，顺便教下小白（PS：我自己也是小白）。

为什么用maven？

用Java框架（SSH、SSM）开发过Web项目的都知道，随着项目越做越大，在WEB-INF/lib目录下放的Jar包也越来越多，当你需要增加或删除一个Jar包，全靠人工记忆，非常不便于管理项目。
而且各个Jar包是否有版本冲突情况，让人头疼。

maven改进

当你需要一个Jar包时，你只需要在pom.xml中申明它，然后更新工程，Jar包会自动下载下来，一般在C:/Users/xx/.m2/repository文件夹中（看你具体怎么配置的）。
当项目编译时，Jar包才会被导入到编译后的项目目录中，随着项目发布。
